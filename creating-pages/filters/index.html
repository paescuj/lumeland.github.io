Creating pages

Filters

Filters are functions that can be applied to variables to transform content. Nunjucks template engine has some builtin filters, for example:

<h1>{{ 'Welcome' | upper }}</h1>

Output:

<h1>WELCOME</h1>

Lume allows to create your own filters to be used by all template engines. New filters must be registered in the _config.js file:

// Filter to prepend a üëç to any text
site.filter("thumbsUp", (value) => "üëç " + value);

Now this filter is available in your layouts:

<h1>{{ 'Welcome' | upper | thumbsUp }}</h1>

Output:

<h1>üëç WELCOME</h1>

Builtin filters

Lume includes the following convenient preinstalled filters:

  • md: It's installed by the markdown plugin and allows to render markdown content to HTML. More info
  • njk: It's installed by the nunjucks plugin and allows to render nunjucks content to HTML. More info
  • url / htmlUrl: It's installed by the url plugin and allows to normalize urls. More info
  • attr / class: It's installed by the attributes plugin and allows to work with HTML attributes. More info

Using the filters in javascript modules

If you're using javascript/typescript modules instead a template engine like Nunjucks, filters are passed as the second argument of your default exported function:

export default function(data, filters) {
  return `<a href="${filters.url('/about-us')}">About us</a>`
}
